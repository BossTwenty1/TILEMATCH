   1.‚Äã INTRODUCTION
   1.1‚ÄãPurpose
              This document describes the requirements of the TileMatch E-Commerce
       Platform. The system allows customers to browse tile products, place orders,
       and pay via GCash, while administrators manage products and orders. The
       system provides a centralized online platform for tile retail, enabling convenient
       purchasing without requiring visits to physical showrooms.

   1.2‚ÄãScope

   Include:

      ‚óè‚Äã   Online product browsing and search: Customers can view tile products
           organized by category, search by keywords, and filter by price, size, material,
           and color to find tiles matching their needs.

      ‚óè‚Äã   Product filtering by category, color, size, material: Advanced filtering allows
           customers to narrow down product selection based on multiple criteria
           simultaneously for efficient shopping.

      ‚óè‚Äã   Shopping cart management: Customers can add and remove items from cart,
           adjust quantities, and save cart contents for later purchase with persistent
           storage across sessions.

      ‚óè‚Äã   Checkout and order placement: Streamlined checkout process where
           customers enter shipping address and review order details before finalizing
           purchase.

      ‚óè‚Äã   GCash payment recording: Payment method captured and recorded in order
           database using GCash payment reference system without live API
           integration.

      ‚óè‚Äã   Order tracking and status updates: Customers can view order history, track
           current order status, and receive email notifications when status changes.

      ‚óè‚Äã   Customer account management: Users can register, login, update profile
           information, and reset forgotten passwords via email.

      ‚óè‚Äã   Admin product management: Store administrators can add, edit, and delete
           tile products; upload images; and manage product categories.

      ‚óè‚Äã   Admin order management: Administrators can view all customer orders,
           update order status, and send status notifications.

      ‚óè‚Äã   Sales reporting and analytics: Dashboard displays key metrics including total
           revenue, order count, best-selling products, and monthly sales trends.

   2.‚Äã OVERALL DESCRIPTION
   2.1 System Overview

The TileMatch system operates as follows:

           Customer registers and logs in ‚Üí browses tile products ‚Üí searches and
   filters by preferences ‚Üí selects product and views details ‚Üí adds items to cart ‚Üí
   proceeds to checkout ‚Üí enters shipping address ‚Üí selects GCash payment ‚Üí
   receives payment reference ‚Üí transfers payment ‚Üí order is confirmed ‚Üí customer
   receives confirmation email ‚Üí customer can track order status ‚Üí admin receives
   order notification ‚Üí admin updates order status ‚Üí customer receives status update
   emails.

   2.2 System Features (High Level)

      ‚óè‚Äã   User registration and login with email verification: Customers create secure
           accounts using email and password, with email confirmation to verify account
           ownership.

      ‚óè‚Äã   Product catalog with tiles organized by category: Complete inventory of tile
           products organized into 5 main categories (Ceramic, Porcelain, Glass,
           Natural Stone, Decorative) for easy browsing.

      ‚óè‚Äã   Advanced search and filtering capabilities: Customers can search by product
           name, color, material, size and apply multiple filters simultaneously to find
           specific tiles.

      ‚óè‚Äã   Shopping cart with persistent storage: Customers can add items to cart, and
           cart contents are saved in the database so items remain even if customer
           logs out.

      ‚óè‚Äã   Checkout with address validation: Secure checkout process that collects and
           validates shipping address information before order confirmation.

      ‚óè‚Äã   GCash payment integration: Payment method using GCash reference system
           where customers receive payment reference number to complete transaction.

      ‚óè‚Äã   Order tracking and history: Customers can view all past orders and track
           current order status with estimated delivery date.

      ‚óè‚Äã   Order status management: Admin updates order status through dashboard
           (Pending ‚Üí Processing ‚Üí Shipped ‚Üí Delivered) with automatic customer
           notifications.

      ‚óè‚Äã   Admin dashboard with overview metrics: Main admin page displaying key
           business metrics including revenue, order count, customer count, and recent
           orders.

      ‚óè‚Äã   Product management (add, edit, delete): Admin interface allowing
           administrators to add new products, modify existing product information, and
           remove discontinued items.
